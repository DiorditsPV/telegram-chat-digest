# Скилл: промоут + деплой фичи (ship)

Финальная фаза флоу: `designed → building → done (кандидат на ветке) → **shipped** (в default-ветке + задеплоено)`.
Решение о выкатке — за человеком; этот скилл его исполняет. **Не часть автономного лупа.**

## 0. Контекст (через `cat`, НЕ Read)
- `cat .claude/feature-loop.md` — секция **Deploy** (`trigger`/`commands`/`checks`), `default_branch`,
  `commit_trailer`. Если секции Deploy нет — спроси у пользователя, как деплоить (или считай `trigger: none`).

## 1. Выбрать фичу и проверить готовность
- Возьми названную пользователем спеку, иначе предложи список `status: done, verify: pass`.
- **Гейт:** ship только если `status: done` и `verify: pass`. Если `building`/`fail` — стоп, верни на `feature-build`.
- Убедись, что ветка `<branch>` существует и содержит код фичи.

## 2. (Опц.) перепроверить ветку
Если со сборки прошло время — переключись на ветку и перепрогони **Verify** из конфига. Красный → не выкатывать.

## 3. Слить в default-ветку
```bash
git switch <default_branch> && git pull --ff-only 2>/dev/null || true
git merge --no-ff <branch> -m "feat(<slug>): <title> (ship)"
```
Конфликты — разрули или верни пользователю. Не теряй чужие коммиты.

## 4. Деплой по `trigger`
- **`merge-to-<default_branch>`** — деплоит CI на мёрж. Сам команды НЕ запускай; дождись/проверь `checks`
  (healthcheck/URL). Если нужен push для срабатывания CI — он разрешён здесь (это явная выкатка), с ретраями.
- **`manual-command`** — выполни `commands` из секции Deploy; затем проверь `checks`.
- **`none`** — деплоя нет (библиотека/локальный инструмент), просто иди дальше.
- Деплой упал — откати мёрж (`git revert -m 1 HEAD` или сбрось), сообщи, не помечай shipped.

## 5. Пометить shipped + каталог (ledger на default-ветке)
- В спеке: `status: shipped` (+ при желании `shipped: <YYYY-MM-DD>`).
- В `FEATURE_IDEAS.md` идея уже `[x]` (после build) — оставь; при желании допиши `(shipped)`.
- `python3 .claude/skills/feature-build/catalog.py` (обновит каталог).
- Коммит ledger (+ `commit_trailer`, если задан). Push — по политике проекта/просьбе пользователя.

## 6. Отчитаться
Кратко: что слито, как задеплоено (CI/команды/none), результат `checks`, новый статус. Если что-то
из этого требует прав/доступов, которых нет — честно скажи, что осталось сделать человеку.
