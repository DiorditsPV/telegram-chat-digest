# AGENTS.md

Полное описание проекта, карта модулей и поток данных — в [`README.md`](README.md)
(см. раздел «Карта проекта (для агентов)»). Здесь — только короткие правила работы.

## Команды (точки входа)
- `/tg-sync` — выгрузить дельту и обновить `context.md` + `delta-log.md` + журнал решений `decisions.jsonl`.
- `/tg-ask <вопрос>` — ответить по накопленной истории.
- `/tg-topics` — доска решений/задач по темам (из `decisions.jsonl`).
- `/tg-why <тема>` — цепочка причина → решение → следствие по теме (с цитатами `#id`).
- `/tg-digest [дней]` — обзор движения решений/задач за период + что зависло.
- `/tg-overview` — обзор активных групп за N дней (флаг `--days`, по умолчанию 7).
- `/tg-discover [N]` — N последних диалогов → `config.discovered.yaml` (кандидаты в цели; файл в `.gitignore`).

Скрипты запускай **питоном из venv**: `.venv/bin/python scripts/<...>.py`
(системный python даст `ModuleNotFoundError: telethon`).

## Инварианты — НЕ нарушать
- **Никогда не коммить реальные данные и секреты:** `.env`, `secrets/`, `*.session`
  и вся `data/` — в `.gitignore`. В git идёт только синтетический `examples/`.
- **`*.jsonl` — append-only источник истины**, нарезка недельными шардами
  `data/<name>/<YYYY-Www>.jsonl` по ISO-неделе даты сообщения (UTC). Не редактируй
  записанное и не возвращай единый `raw.jsonl`.
- **Тянем только новые сообщения** (дельта по `min_id`); правки/удаления/реакции не
  догоняем — учитывай это при ответах.
- **`context.md` мерджится, не переписывается:** старые решения не удаляй, при смене
  статуса обновляй соответствующую строку.
- **`decisions.jsonl` — append-only event-log** решений/задач по темам: изменение статуса
  фиксируй НОВЫМ событием с тем же `item_id` (не редактируй прошлые строки). Логика
  свёртки/рендера — в `scripts/decisions.py`; формат — в `examples/team-platform/decisions.jsonl`.

## Перед изменением кода
- Формат записи `*.jsonl` и артефактов — в `examples/team-platform/`; держи единый
  формат между `tg_sync.py`, `import_export.py` и `migrate_to_weekly.py`.
- Логика ISO-недели и недельного шардинга (`iso_week_of`, бакетизация, дозапись) живёт в одном
  модуле `scripts/shards.py`; `tg_sync.py`, `import_export.py`, `migrate_to_weekly.py` импортируют
  его (`import shards`). Не возрождай локальные копии — меняй в `shards.py`.
